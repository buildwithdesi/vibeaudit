## CRITICAL Findings

### 1. Client-Side AI API Key Exposure — `digital-alchemy-os`

| Field | Value |
|-------|-------|
| Rule | `exposed-secrets` / `client-side-trust` |
| CWE | CWE-798 (Hardcoded Credentials) |
| CVSS | 9.1 |
| File | `src/lib/gemini.ts` |

```ts
const API_KEY = process.env.NEXT_PUBLIC_GEMINI_API_KEY || "";
```

`NEXT_PUBLIC_` prefix ships this key to every browser visitor. Anyone can extract it from DevTools → Network tab and call Google AI on your billing account. **This is the highest-priority fix.**

**Fix:** Move the Gemini call to a server-side API route (`/api/generate`). Remove the `NEXT_PUBLIC_` prefix. The client should call your API route, not Google directly.

### 2. Unprotected API Routes Calling Paid AI Services (3 repos, 11+ routes)

| Field | Value |
|-------|-------|
| Rule | `missing-auth` / `missing-rate-limiting` / `ai-cost-exposure` |
| CWE | CWE-306 (Missing Authentication), CWE-770 (No Rate Limit) |
| CVSS | 7.5 |

**Affected repos and routes:**

| Repo | Unprotected Routes | Service Called |
|------|-------------------|---------------|
| `VIDEOANALYZERGOOGLE` | `api/generate.ts`, `api/upload/presign.ts` | Gemini + R2 |
| `video-analyzer-70` | `api/generate.ts`, `api/upload.ts`, `api/classify.ts`, `api/generate-chunked.ts`, `api/upload-chunk.ts`, `api/init-upload.ts`, `api/files/[name].ts` | Gemini |
| `vibeshot2` | `api/phase1.ts`, `api/phase2.ts` | Gemini |

All routes accept unauthenticated requests and forward them to Google's Gemini API. A bot can discover these endpoints and run up your API bill.

**Fix:** Add auth middleware (Clerk `getAuth()`, Supabase `getUser()`, or a simple API key check). Add rate limiting via Upstash or Vercel KV.
