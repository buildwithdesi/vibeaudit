### ADVISORY (3 items)

#### 1. Rate limiting only present in 2 of ~30 repos with API routes

**Repos with rate limiting:**
- `homiedex` - Upstash Redis sliding window rate limiter
- `video-analyzer-70` - express-rate-limit
- `content-drop` - IP throttle in middleware (well-implemented)

**Repos with API routes but no visible rate limiting:**
- Most other repos with `/api/` routes (the-77, Content-Hook-Generator, vibeshot2, node-banana-bear, chibi-forge, VIDEOANALYZERGOOGLE, etc.)

**Risk:** Without rate limiting, AI-powered API endpoints (Gemini, OpenAI) can be abused to run up provider bills. Most of these appear to be demos/experiments, but any deployed one is exposed.

**Fix:** Add rate limiting to any deployed API route that calls a paid AI service. For Vercel: `@upstash/ratelimit`. For Express: `express-rate-limit`.

#### 2. No security headers (helmet/CSP) detected in any application repo

**Risk:** Missing `Content-Security-Policy`, `X-Frame-Options`, `Strict-Transport-Security` headers. Low severity for most of these apps, but relevant for any handling auth or payments.

**Repos where this matters most:** content-drop, digitalalchemy-dev, homiedex, Storefront, saas-starter

**Fix:** For Next.js apps, add headers in `next.config.js` or middleware. For Express: `npm install helmet`.

#### 3. ~120 dormant repos with no recent activity

161 total repos, but only ~40 have been touched since April 2026. The rest are experiments from the 100-day challenge and earlier work. While not a security issue per se, dormant repos with outdated dependencies could become a vector if any are still deployed.

**Recommendation:** Archive repos that are no longer deployed or maintained.

## Scan Limitations

1. **No GITHUB_TOKEN configured** - vibeaudit's built-in GitHub API scanner was rate-limited. Scan used MCP GitHub code search instead, which has its own auth but different search capabilities.
2. **Code search scope** - GitHub code search indexes file contents but may not cover all files (binary, very large, or recently pushed files can lag).
3. **No dependency audit (SCA)** - `npm audit` / `pip audit` requires local clones. Not performed in this scan.
4. **No git history secrets scan** - Would require `--deep` flag with local clones.

**Recommendation for next scan:** Set `GITHUB_TOKEN` as an environment variable so vibeaudit can run its full 85-rule AST-based analysis directly against each repo via the GitHub API.
